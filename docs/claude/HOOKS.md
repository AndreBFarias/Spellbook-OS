# docs/claude/HOOKS.md — Hooks do Claude Code

3 hooks customizados (mantidos neste repo) + 4 hooks vindos de plugins.

## Hooks customizados (em `docs/claude/hooks/`)

### 1. `guardian.py` — PreToolUse anti-emoji / anti-IA

**Matcher**: `Write|Edit|MultiEdit`
**Timeout**: 5s

Bloqueia conteúdo que viola CLAUDE.md:
- **Emojis** (regex unicode ranges clássicos — smileys, símbolos, transport, dingbats)
- **Atribuições a IA**: `Co-Authored-By|Generated by|Created by|Written by|Paired-With|Assisted-By` acoplado a nome de ferramenta de IA
- **Endereços de atribuição** em commits (emails genéricos usados por ferramentas de IA)

**Paths isentos** (onde regex são legítimos por natureza):
- `/.claude/hooks/guardian.py`
- `/_lib.sh`, `/emoji_guardian.py`, `/sanitizar_ia.py`, `/universal-sanitizer.py`
- `/CLAUDE.md`, `/AI.md`

**Retorno de bloqueio**: `{"decision": "block", "reason": "<motivo + contexto 30 chars>"}`.

### 2. `session-start-briefing.py` — SessionStart capacitado

**Matcher**: `*` (qualquer origem — startup, resume, clear, compact)
**Timeout**: 10s

Coração do "boot capacitado". Lê variáveis de ambiente exportadas pelo `cca` + arquivos leves + emite `additionalContext` JSON no stdout.

**Leitura** (todas best-effort, timeout 500ms cada):
- Variáveis: `$CLAUDE_PROJECT_ROOT`, `$CLAUDE_PROJECT_NAME`, `$CLAUDE_BRIEF_PATH`, `$CLAUDE_PROJECT_KIND`, `$CLAUDE_BRIEF_STATUS`
- `$CLAUDE_BRIEF_PATH` (status + linhas + idade)
- `$CLAUDE_PROJECT_ROOT/CLAUDE.md` (se local existir)
- `~/.claude/templates/bootstrap-<kind>.md` (se kind conhecido)
- `SPECIAL_PROJECTS.json` (mapa canônico)

**Checks de saúde**:
- `test -x ~/.claude/chrome/chrome-native-host` (extensão Chrome pareada?)
- `test -d ~/.config/google-chrome/Default/Extensions/fcoeoabgfenejglbffodgkkbkcdhcgfn/1.0.68_0`
- `test -d ~/.npm/_npx/*/node_modules/@playwright/mcp` (cache do playwright)

**Emite** (JSON no stdout):
```json
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "[SANTUÁRIO READY]...\n[PROJETO ESPECIAL]...\n[CAPACIDADES VISUAIS]...\n[SPRINT CICLO]...\n[AÇÃO AUTOMÁTICA]..."
  }
}
```

**Se BRIEF ausente + projeto conhecido com memórias**: bloco `[AÇÃO AUTOMÁTICA]` instrui Claude a auto-disparar `python3 ~/.config/zsh/scripts/bootstrap-rico-brief.py --projeto <kind>` antes de qualquer tarefa.

**Se BRIEF ausente + genérico**: instrui auto-dispatch de `validador-sprint` em MODO BOOTSTRAP.

**Sempre `exit 0`** — qualquer erro vira linha informativa no additionalContext.

### 3. `post-plan-clear.py` — UserPromptSubmit sutil

**Matcher**: `*`
**Timeout**: 5s

Detecta triggers: `aprovo|aprovado|pode executar|exit plan mode|segue|implementa`.

Se detecta, emite additionalContext SUGERINDO (não forçando) `/clear` ou uso de subagents: "DICA: Plano aparentemente aprovado. Para executar com contexto fresco, prefira dispatch executor-sprint (subagent isolado) ou `/clear` manual. O ciclo `/sprint-ciclo` já resolve isso automaticamente."

Mitigação para quando o menu nativo "Yes, clear context and auto-accept edits" não aparece (bug #45034 / #38071 / #39665 do Claude Code).

## Hooks de plugins (auto-carregados)

Vêm de `~/.claude/plugins/cache/claude-plugins-official/<plugin>/hooks/hooks.json` e são registrados automaticamente pelo harness quando o plugin está em `enabledPlugins`.

### `superpowers` — SessionStart
**Matcher**: `startup|clear|compact`
**Comando**: `${CLAUDE_PLUGIN_ROOT}/hooks/run-hook.cmd session-start`
**Efeito**: injeta `using-superpowers/SKILL.md` no contexto inicial.

### `learning-output-style` — SessionStart
**Matcher**: `*`
**Comando**: `bash ${CLAUDE_PLUGIN_ROOT}/hooks-handlers/session-start.sh`
**Efeito**: ativa output style "learning" com instruções pedagógicas.

### `hookify` — PreToolUse, PostToolUse, Stop, UserPromptSubmit
**Matcher**: `*` em cada evento (timeout 10s)
**Comandos**: `python3 ${CLAUDE_PLUGIN_ROOT}/hooks/{pretooluse,posttooluse,stop,userpromptsubmit}.py`
**Efeito**: executa regras locais configuradas via `/hookify` commands.

### `security-guidance` — PreToolUse
**Matcher**: `Edit|Write|MultiEdit`
**Comando**: `python3 ${CLAUDE_PLUGIN_ROOT}/hooks/security_reminder_hook.py`
**Efeito**: injeta lembretes de segurança em edits sensíveis.

## Ordem de execução em sessão nova

```
claude boot
    |
    +-- SessionStart (paralelo, não-bloqueante):
    |     superpowers/session-start         -> contexto de superpowers
    |     learning-output-style/session-start -> ativa modo learning
    |     session-start-briefing.py (custom) -> additionalContext do projeto
    |
    +-- primeira mensagem do usuário:
    |     UserPromptSubmit (paralelo):
    |       hookify/userpromptsubmit.py     -> regras locais
    |       post-plan-clear.py (custom)     -> sugere /clear se aprovação
    |
    +-- Edit/Write/MultiEdit:
    |     PreToolUse (paralelo):
    |       hookify/pretooluse.py           -> regras locais
    |       security-guidance/...           -> lembrete de segurança
    |       guardian.py (custom)            -> anti-emoji/anti-IA
    |
    +-- (tool use termina):
    |     PostToolUse (paralelo):
    |       hookify/posttooluse.py          -> regras locais
    |
    +-- Stop:
          hookify/stop.py                   -> regras locais
```

Cada hook tem stdout/additionalContext isolado. Não há conflito entre eles. Performance: startup total < 3s em p99.

## Troubleshooting

### Hook não dispara
1. `claude --debug` — ver no stdout qual hook foi chamado
2. `ls -la ~/.claude/hooks/*.py` — symlink está OK?
3. `python3 ~/.claude/hooks/session-start-briefing.py < /dev/null` — roda standalone?

### Hook bloqueia algo legítimo
1. Path é isento? Adicionar em `guardian.py` na lista `IGNORE_IF_PATH_CONTAINS`.
2. Violação real? Corrigir o conteúdo.

### SessionStart demora
1. Cada hook tem timeout individual. Se um hook trava, o harness mata após timeout.
2. Pode-se desabilitar temporariamente removendo entry de `settings.json`.
